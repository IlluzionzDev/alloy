  ### Remarks

This component must be called in one of two ways: with a name and an optional refkey, or else passing in the symbol. When called with a name and refkey, a symbol will be created in the current scope with that name and refkey. If a refkey is not provided, `refkey(props.name)` is used.

When called with a symbol, that symbol is merely exposed via <a href="../../contexts/declaration/">Declaration context</a>. It is assumed that the caller of this component has created the symbol with the `createSymbol` API on the <a href="../../contexts/binder/">Binder context</a>.
